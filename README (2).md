# Граница эффективности посттренировочной квантизации: исследование на семействе Qwen2.5

## Аннотация

Работа посвящена систематическому изучению деградации качества языковых моделей при применении посттренировочной квантизации (PTQ) в формате GGUF на моделях семейства Qwen2.5 с числом параметров от 0.5B до 14B. Основная исследовательская гипотеза состоит в том, что существует критический порог числа параметров, ниже которого агрессивная квантизация приводит к значимой (>5%) деградации качества. Результаты подтверждают гипотезу и позволяют количественно охарактеризовать границы безопасного применения каждого формата квантизации.

---

## Постановка задачи

Квантизация позволяет снизить требования к памяти при инференсе, однако сопровождается потерей качества. Для малых моделей (менее 7B параметров) эта потеря может оказаться неприемлемой. Настоящее исследование отвечает на вопрос: **при каком числе параметров и каком формате квантизации деградация perplexity превышает порог в 5%?**

## Результаты

### Perplexity на WikiText-2 (baseline = Q8_0)

| Модель | Q8\_0  | Q5\_K\_M        | Q4\_K\_M        | Q3\_K\_M          | Q2\_K                    |
|--------|--------|-----------------|-----------------|-------------------|--------------------------|
| 0.5B   | 13.12  | 13.36 (+1.8%)   | 13.55 (+3.3%)   | 14.17 (+8.0%)     | 15.76 (+20.1%)           |
| 1.5B   | 9.20   | 9.32  (+1.3%)   | 9.50  (+3.2%)   | 10.70 (+16.3%)    | 21.82 (+137.2%)          |
| 3B     | 7.95   | 8.13  (+2.3%)   | 8.35  (+5.0%)   | 11.65 (+46.5%)    | 21454.95 (коллапс)       |
| 7B     | 6.74   | 6.82  (+1.2%)   | 6.87  (+1.9%)   | 7.25  (+7.6%)     | 9.44  (+40.1%)           |
| 14B    | 5.11   | —               | —               | —                 | 7.43  (+45.4%)\*         |

\* Деградация 14B Q2\_K вычислена относительно fp16 baseline, полученного через HF Transformers (PPL = 5.81).

Прочерки в столбцах Q4\_K\_M, Q5\_K\_M и Q3\_K\_M для модели 14B обусловлены техническим ограничением файловой системы Lustre на кластере ВШЭ: запись промежуточного GGUF-файла размером 29.5 ГБ завершается с ошибкой квоты до окончания записи.

### Границы безопасного применения квантизации

| Формат   | Граница | Обоснование                                                                  |
|----------|---------|------------------------------------------------------------------------------|
| Q5\_K\_M | Все размеры | Деградация не превышает 2.3% ни на одной из исследованных моделей        |
| Q4\_K\_M | >= 7B       | На 3B деградация составляет +5.0% (пороговый случай); на 7B — +1.9%      |
| Q3\_K\_M | >= 7B       | На 3B — +46.5%; на 7B — +7.6%; на 1.5B — +16.3%                          |
| Q2\_K    | Неприменим  | Коллапс на 3B (PPL = 21 454); критическая деградация на всех размерах     |

### Ключевые наблюдения

**1. Коллапс модели Qwen2.5-3B при Q2\_K.**
Значение perplexity 21 454 при baseline 7.95 свидетельствует о полной утрате когерентности генерации. Модель фактически неработоспособна. Данный эффект не наблюдается ни на 1.5B (PPL = 21.82, деградация +137%), ни на 7B (PPL = 9.44, деградация +40%), что указывает на нелинейный, скачкообразный характер деградации в диапазоне 1.5B–3B для формата Q2\_K.

**2. Нелинейность деградации Q3\_K\_M в диапазоне 1.5B–3B.**
Переход от 1.5B (+16.3%) к 3B (+46.5%) при Q3\_K\_M демонстрирует резкий рост деградации, несопоставимый с плавным улучшением от 3B к 7B (+7.6%). Это указывает на существование критического порога между 1.5B и 3B, ниже которого Q3\_K\_M становится непригодным.

**3. Q4\_K\_M как оптимальный формат для моделей >= 7B.**
На моделях 7B и 14B деградация Q4\_K\_M не превышает 1.9% при двукратном сокращении размера модели относительно Q8\_0. Соотношение качество/компрессия является оптимальным для практического применения.

**4. Q5\_K\_M как универсально безопасный формат.**
Деградация Q5\_K\_M не превышает 2.3% на всём исследованном диапазоне размеров. Данный формат рекомендуется при строгих требованиях к качеству для моделей любого размера.

## Методология

### Модели

Исследование проводилось на семействе Qwen2.5 (Qwen/Qwen2.5-{0.5,1.5,3,7,14}B). Использование одного семейства моделей позволяет исключить влияние архитектурных различий и изолировать эффект масштаба.

### Квантизация

Конвертация моделей из формата HuggingFace SafeTensors в GGUF и последующая квантизация выполнялись с использованием llama.cpp (build 9210, commit c3f95c1f0, GNU 12.1.0). Исследовались форматы Q2\_K, Q3\_K\_M, Q4\_K\_M, Q5\_K\_M, Q8\_0.

### Оценка качества

Метрика: perplexity на тестовой выборке WikiText-2 (wikitext-2-raw-v1, split=test).
Параметры оценки: длина контекста 512 токенов, шаг сдвига (stride) 512 токенов, 16 потоков CPU.
Инструмент: `llama-perplexity` из состава llama.cpp.

Baseline: Q8\_0 используется как proxy для fp16, поскольку конвертация fp16 GGUF для 14B (29.5 ГБ) завершается с ошибкой записи на файловой системе Lustre. fp16 baseline для 14B получен отдельно через HF Transformers (float32, CPU) и составил PPL = 5.81.

### Инфраструктура

Эксперименты выполнены на кластере ВШЭ cHARISMa, узлы типа A/B (Tesla V100 SXM2 32 ГБ, 4 GPU на узел). Квантизация и оценка perplexity выполнялись на CPU (44 ядра, до 663 ГБ RAM).

---

## Ограничения

- Q4\_K\_M, Q5\_K\_M и Q3\_K\_M для модели 14B недоступны из-за ограничений файловой системы Lustre при записи файлов размером >28 ГБ.
- fp16 baseline через llama-perplexity получен только для 7B; для остальных моделей в качестве baseline используется Q8\_0, что вносит систематическую погрешность порядка 0.5–1%.
- Оценка проводилась на одном датасете (WikiText-2, английский язык). Результаты могут отличаться для других доменов (код, математика, нерелигиозные языки).
- Один прогон на каждую модель без оценки дисперсии.
- Исследование охватывает только архитектуру Qwen2.5; выводы могут не переноситься на другие семейства (Llama, Gemma, Mistral).

## Воспроизведение

### Зависимости

```
llama.cpp >= build 9210
Python >= 3.10
torch, transformers, datasets, huggingface_hub
```

### Окружение (кластер ВШЭ cHARISMa)

```bash
module load gnu12/12.1
module load CUDA/12.4
module load Python/PyTorch_GPU_v2.4
export LD_LIBRARY_PATH=$(dirname $(gcc --print-file-name=libstdc++.so.6)):$LD_LIBRARY_PATH
```

### Шаги воспроизведения

```bash
# 1. Загрузка моделей
python3 -c "
from huggingface_hub import snapshot_download
for size in ['0.5B', '1.5B', '3B', '7B', '14B']:
    snapshot_download(f'Qwen/Qwen2.5-{size}', local_dir=f'./hf_models/Qwen2.5-{size}',
                      ignore_patterns=['*.msgpack', '*.h5', 'flax_model*'])
"

# 2. Сборка llama.cpp (CPU)
git clone https://github.com/ggml-org/llama.cpp
cd llama.cpp && cmake -B build_cpu -DGGML_CUDA=OFF && cmake --build build_cpu -j8

# 3. Конвертация и квантизация (через Slurm)
sbatch scripts/quantize_all.sh

# 4. Оценка perplexity (через Slurm)
sbatch scripts/eval_ppl.sh
```
